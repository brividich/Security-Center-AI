Overview

Security Center AI is a report intelligence portal. It turns reports, emails, and uploads into parsed metrics, findings, alerts, Evidence Containers, remediation tickets, dashboards, admin configuration, and diagnostics.

It is not a full SIEM. It focuses on recurring operational reports and notifications rather than raw high-volume log search.

First Setup Checklist

  • Run python manage.py seed_security_center_config.
  • Review sources, parsers, alert rules, suppressions, backup expectations, notifications, ticketing, and audit log.
  • Open diagnostics and resolve warnings.
  • Ingest known-good WatchGuard, Defender, and Backup/NAS samples.
  • Run parsers, rules, and KPI snapshots from the pipeline page.

Core Workflow

Report/email/upload -> Ingestion -> Parser Engine -> Metrics/Findings -> Rule Engine -> Alert/Suppression -> Evidence Container -> Ticket -> Dashboard/Admin

Addons

Core is the shared engine. An addon is source-specific parsers, seed config, rules, metrics, dashboards, tests, and docs. Current addons: WatchGuard, Microsoft Defender, and Backup/NAS.

Admin Configuration

Use admin config for sources, parsers, alert rules, suppressions, backups, notifications, ticketing, and audit log.

Diagnostics

Use diagnostics after setup, parser changes, seed changes, notification changes, and before relying on a new source.

Troubleshooting

  • No parser matched: check source patterns, parser name, parser enabled state, and diagnostics.
  • No alert created: check enabled rules, metric names, thresholds, cooldown, deduplication, and suppressions.
  • No ticket created: check auto-ticket settings and whether an existing ticket was updated.
  • Backup missing not detected: check expected job name, schedule, missing-after hours, and last seen status.
  • Permission denied: use staff access or the manage_security_configuration permission.

Operations Checklist

  • Daily: dashboard, critical alerts, Defender CVE tickets, failed or missing backups, parser warnings.
  • Weekly: suppressions, noisy rules, backup trends, unresolved tickets.
  • Monthly: thresholds, evidence retention, KPI trends, audit logs, management reporting.
