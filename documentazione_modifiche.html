📋 Documentazione Modifiche Recenti

Security Center AI - Security Hardening & UX Improvements

🎯 Panoramica

Le modifiche rappresentano un importante hardening di sicurezza del progetto Security Center AI, con miglioramenti significativi sia backend che frontend.

🔒 Backend - Security Hardening

1. Validazione SECRET_KEY Obbligatoria

security_center_ai/settings/base.py:96-99

if not SECRET_KEY: raise ValueError("SECRET_KEY environment variable is required")

Cosa: Previene deployment accidentale con chiavi di default.

Perché: Evita configurazioni insicure in produzione.

2. Security Headers Comprehensivi

security_center_ai/settings/base.py:184-197

Headers aggiunti:

  • HSTS (1 anno, subdomains, preload)
  • X-Frame-Options: DENY
  • X-Content-Type-Options: nosniff
  • X-XSS-Protection: 1; mode=block
  • Secure cookie settings
  • Strict referrer policy

Perché: Protezione contro clickjacking, XSS, e altri attacchi.

3. Validazione Input Robusta

security/views.py:104-122

Miglioramenti:

  • Validazione severity contro whitelist
  • Validazione date con try/except
  • Validazione numeri con isdigit()
❌ PRIMA
if request.GET.get("severity"): alerts = alerts.filter(severity=request.GET["severity"])
✅ DOPO
severity = request.GET.get("severity") if severity and severity in [s[0] for s in Severity.choices]: alerts = alerts.filter(severity=severity)

Perché: Previene injection e errori di validazione.

4. CSRF Protection Completa

security/views.py

Cosa: Aggiunto @ensure_csrf_cookie a TUTTE le views (15+ endpoint).

Perché: Protezione CSRF su tutti gli endpoint API.
